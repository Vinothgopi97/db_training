Database Relationships

Types of Relationships

1. One-to-One (1:1)

Each record in one table is related to exactly one record in another table.

CREATE TABLE users (
id SERIAL PRIMARY KEY,
username VARCHAR(50)
);

CREATE TABLE user_profiles (
id SERIAL PRIMARY KEY,
user_id INTEGER UNIQUE REFERENCES users(id),
bio TEXT
);

2. One-to-Many (1:N)

One record in a table can be related to multiple records in another table.

CREATE TABLE authors (
id SERIAL PRIMARY KEY,
name VARCHAR(100)
);

CREATE TABLE books (
id SERIAL PRIMARY KEY,
title VARCHAR(200),
author_id INTEGER REFERENCES authors(id)
);

3. Many-to-Many (N:M)

Multiple records in one table can be related to multiple records in another table.

CREATE TABLE students (
id SERIAL PRIMARY KEY,
name VARCHAR(100)
);

CREATE TABLE courses (
id SERIAL PRIMARY KEY,
title VARCHAR(200)
);

CREATE TABLE student_courses (
student_id INTEGER REFERENCES students(id),
course_id INTEGER REFERENCES courses(id),
PRIMARY KEY (student_id, course_id)
);

Implementing Relationships

  • Foreign Keys: Used to establish relationships between tables
  • Junction Tables: Used for many-to-many relationships
  • Cascading Actions: Automatic actions on related records

Cascading Actions

CREATE TABLE orders (
id SERIAL PRIMARY KEY,
user_id INTEGER REFERENCES users(id)
ON DELETE CASCADE
ON UPDATE CASCADE,
order_date DATE
);

Best Practices

  • Choose the appropriate relationship type based on business rules
  • Use meaningful names for foreign key columns
  • Consider referential integrity constraints
  • Plan for data modification scenarios
  • Consider indexing foreign key columns

Common Pitfalls to Avoid:

  • Over-normalizing tables
  • Creating unnecessary relationships
  • Ignoring referential integrity
  • Not considering the impact on performance
