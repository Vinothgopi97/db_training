The Relational Model

Understanding the Relational Model

The relational model, developed by E.F. Codd in 1970, represents data as relations or tables. Each relation consists of tuples (rows) and attributes (columns).

Key Components

  • Relations (Tables): A set of tuples that have the same attributes
  • Tuples (Rows): A single record in a relation
  • Attributes (Columns): The properties or fields that define a relation
  • Domain: The set of allowable values for an attribute

Properties of Relations

  • Atomic Values: Each cell contains exactly one value
  • Unique Rows: No duplicate tuples allowed
  • Column Order Irrelevant: The order of columns doesn't matter
  • Row Order Irrelevant: The order of rows doesn't matter

Keys in Relational Model

  • Primary Key: Uniquely identifies each tuple in a relation
  • Foreign Key: References a primary key in another relation
  • Candidate Key: Attributes that could serve as a primary key
  • Composite Key: Multiple attributes together forming a unique identifier

Database Normalization:
The process of organizing data to minimize redundancy and dependency by:

  • Eliminating duplicated data
  • Ensuring data dependencies make sense
  • Making the database more efficient
