SQL Basics

What is SQL?

SQL (Structured Query Language) is the standard language for managing and manipulating relational databases. It's used for storing, manipulating, and retrieving data in databases.

Basic SQL Commands

DDL (Data Definition Language)

-- Create a table
CREATE TABLE users (
id INT AUTO_INCREMENT PRIMARY KEY,
name VARCHAR(100),
email VARCHAR(255)
);

-- Modify table structure
ALTER TABLE users ADD COLUMN age INTEGER;

-- Delete table
DROP TABLE users;

DML (Data Manipulation Language)

-- Insert data
INSERT INTO users (name, email) VALUES ('John', 'john@example.com');

-- Update data
UPDATE users SET age = 25 WHERE name = 'John';

-- Delete data
DELETE FROM users WHERE name = 'John';

DQL (Data Query Language)

-- Select all columns
SELECT * FROM users;

-- Select specific columns
SELECT name, email FROM users;

-- Filter results
SELECT * FROM users WHERE age > 21;

-- Sort results
SELECT * FROM users ORDER BY name ASC;

SQL Operators — Reference and Examples

Operators are the building blocks of SQL expressions. Below are the common operator categories you will use in MySQL.

Comparison operators

  • = — equality (NULL-safe equality uses IS / IS NOT for NULL checks)
  • <> or != — not equal
  • <, <=, >, >= — less/greater than
  • BETWEEN a AND b — inclusive range check
  • IN (a, b, c) — membership in a list
SELECT * FROM users WHERE age BETWEEN 18 AND 30; SELECT * FROM products WHERE category IN ('books','games');

Logical operators

Combine boolean conditions using:

  • AND — both conditions true
  • OR — at least one true
  • NOT — negation
SELECT * FROM orders WHERE status = 'paid' AND total > 0;

Arithmetic operators

Use for numeric math: +, -, *, /, % (modulo).

UPDATE accounts SET balance = balance - 100 WHERE id = 1;

String and pattern operators

  • LIKE — pattern match with % and _ wildcards.
  • REGEXP — regular-expression match.
SELECT * FROM users WHERE email LIKE '%@example.com';

NULL handling

NULL is not a value — comparisons with NULL return unknown. Use:

  • IS NULL / IS NOT NULL to test for NULL.
  • COALESCE(expr1, expr2, ...) to provide a default when NULL.
SELECT COALESCE(phone,'(not provided)') FROM contacts;

Bitwise operators

Operate on integers at the bit-level: & (AND), | (OR), ^ (XOR), ~ (NOT), shift << and >>.

JSON operators (MySQL)

MySQL provides JSON-specific operators/functions: ->, ->>, JSON_EXTRACT(), JSON_UNQUOTE(). Example:

SELECT metadata->>'$.user.ip' FROM events WHERE JSON_CONTAINS(metadata,'"mobile"','$.device');

Operator precedence and parentheses

Use parentheses (...) to make evaluation order explicit. Logical AND has higher precedence than OR.

SQL Functions — Reference and Examples

Functions transform data and return values. Below are key function categories with examples commonly used in MySQL.

Aggregate functions

  • COUNT(expr) — row count (use COUNT(*) or COUNT(DISTINCT col)).
  • SUM(expr) — sum of values.
  • AVG(expr) — average of values.
  • MIN(expr), MAX(expr) — minimum/maximum.
SELECT category, COUNT(*) AS cnt, AVG(price) FROM products GROUP BY category;

String functions

  • CONCAT(a,b,...) — concatenate strings.
  • SUBSTRING(s,pos,len) — extract substring.
  • LEFT/RIGHT(s,n) — left/right substring.
  • LOWER/UPPER(s) — case conversion.
  • TRIM(), REPLACE(), FORMAT() etc.
SELECT CONCAT(first_name,' ',last_name) AS full_name FROM users;

Date/time functions

  • NOW(), CURRENT_TIMESTAMP() — current datetime.
  • DATE(expr), TIME(expr), YEAR(expr), MONTH(expr), DAY(expr)
  • DATE_ADD(date, INTERVAL n unit), DATE_SUB() for arithmetic.
  • TIMESTAMPDIFF(unit, datetime1, datetime2) — difference between times.
SELECT DATE(created_at) AS created_date, COUNT(*) FROM orders GROUP BY created_date;

Numeric functions

  • ROUND(x, d), CEIL(x), FLOOR(x)
  • ABS(x), POW(x,y), SQRT(x)

Conditional functions

  • IF(expr, true_val, false_val) — ternary-style conditional.
  • CASE WHEN ... THEN ... [ELSE ...] END — SQL-standard conditional expression.
  • COALESCE(a,b,...) — first non-NULL value.
SELECT CASE WHEN stock < 1 THEN 'out' WHEN stock < 10 THEN 'low' ELSE 'ok' END FROM products;

JSON functions

MySQL exposes many JSON helpers: JSON_EXTRACT(), JSON_UNQUOTE(), JSON_CONTAINS(), JSON_ARRAYAGG(), JSON_OBJECT(). Use generated columns and indexes for performant JSON queries.

Window functions (analytic)

MySQL supports window functions (since 8.0): ROW_NUMBER(), RANK(), SUM() OVER (PARTITION BY ... ORDER BY ...), etc.

SELECT user_id, amount, SUM(amount) OVER (PARTITION BY user_id ORDER BY created_at) AS running_total FROM payments;

Performance & best practices

  • Apply functions to indexed columns carefully — functions on the left side of comparisons can prevent index usage.
  • Prefer sargable expressions in WHERE clauses (e.g., col >= ? rather than YEAR(col) = ?).
  • Use appropriate aggregate grouping and avoid SELECT * in large queries when not needed.

Best Practices:

  • Use meaningful table and column names
  • Always use proper data types
  • Include appropriate constraints
  • Write clear and maintainable queries
  • Use indexes for better performance
