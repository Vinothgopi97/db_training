Database Constraints

What are Constraints?

Constraints are rules enforced on data columns in tables. They are used to limit the type of data that can be stored in a table, ensuring accuracy and reliability of the data.

Types of Constraints

1. PRIMARY KEY Constraint

CREATE TABLE users (
id SERIAL PRIMARY KEY,
username VARCHAR(50) NOT NULL
);

Ensures that a column or combination of columns uniquely identifies each row in a table.

2. FOREIGN KEY Constraint

CREATE TABLE orders (
order_id SERIAL PRIMARY KEY,
user_id INTEGER REFERENCES users(id),
order_date DATE
);

Creates a link between data in two tables, maintaining referential integrity.

3. UNIQUE Constraint

CREATE TABLE users (
id SERIAL PRIMARY KEY,
email VARCHAR(255) UNIQUE,
username VARCHAR(50)
);

Ensures all values in a column are different.

4. NOT NULL Constraint

CREATE TABLE products (
id SERIAL PRIMARY KEY,
name VARCHAR(100) NOT NULL,
price DECIMAL NOT NULL
);

Ensures a column cannot have NULL value.

5. CHECK Constraint

CREATE TABLE products (
id SERIAL PRIMARY KEY,
name VARCHAR(100) NOT NULL,
price DECIMAL CHECK (price > 0)
);

Ensures all values in a column satisfy certain conditions.

Adding Constraints to Existing Tables

-- Add a unique constraint
ALTER TABLE users ADD CONSTRAINT unique_email UNIQUE (email);

-- Add a foreign key constraint
ALTER TABLE orders
ADD CONSTRAINT fk_user
FOREIGN KEY (user_id) REFERENCES users(id);

-- Add a check constraint
ALTER TABLE products
ADD CONSTRAINT positive_price CHECK (price > 0);

Best Practices:

  • Always use appropriate constraints to maintain data integrity
  • Name constraints meaningfully for easier maintenance
  • Consider the impact on performance
  • Plan constraints during database design phase
